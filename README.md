## What is it, in plain English?

LocaLedger is a free app that lives on your phone and tracks how much money you've made — by shift, by location, by week, or by anything else you care about. Nothing syncs to a server. No login. No subscription. No ads. Everything stays in your phone, period.

Perfect if you:
- Work multiple jobs or gigs (DoorDash + a day job, two part-time jobs, freelance clients...)
- Get paid cash and want to keep a record
- Want to know "how much did I actually make this month?" without opening a spreadsheet
- Don't trust apps that want to read your bank account

## What can it actually do?

| Feature | Details |
|---|---|
| ⏱ **Log a shift** | Pick a location, enter hours + rate (or just a dollar amount), add a note — done in under 10 seconds |
| 📍 **Multiple locations** | Color-code each job or client — Uber, Café, Freelance, whatever |
| 🗂 **Custom fields** | Add your own fields to any entry: mileage, tips, job number, client name — any type (text, number, money, checkbox, date) |
| 📊 **Charts** | Pie and bar breakdowns by location, filterable by week / month / year |
| 🔍 **History** | Search and scroll every shift, edit or delete anything, soft-delete with restore |
| 💾 **Backup & Restore** | Export your entire data set as JSON, import it on any other device |
| 🧹 **Data Health** | Scans for duplicate IDs, missing required fields, and bad values — one tap to fix everything |
| 🔄 **Auto-updates** | Settings → App Version → "Check for Updates" — pulls the latest code without touching your data |
| ✈️ **Fully offline** | Works with zero internet after the first load — it's a PWA, cached on your device |
| 🔒 **No accounts** | Literally zero — no email, no password, no OAuth, no cloud |

## How to log your first shift

Once it's on your home screen, open it up:

1. **Tap the `+` (Add) tab** at the bottom
2. **Enter your hours and rate** — or just a total dollar amount
3. **Pick or type a location** — first time? Type a name like "Starbucks" and it'll save it
4. Optionally add a note or any custom fields you've set up
5. **Tap "Save"** — it's in your history immediately, no internet needed

That's it. No account, no confirmation email, no onboarding flow.

## How updating works

LocaLedger can update itself without you deleting and reinstalling. Your data is **never touched** during an update — it lives in your phone's local storage, completely separate from the app code.

```
Settings tab → "App Version" card → "Check for Updates" → "Update Now"
```

The app compares your installed version against the deployed `version.json`. If there's something new, one tap reloads the app with fresh code. Your shifts, locations, and custom fields are all still there.

## Tech stack (for the curious)

LocaLedger is intentionally a zero-dependency, single-file web app. There's no build step, no npm in production, no framework to update.

| Layer | What's used | Why |
|---|---|---|
| **UI** | Vanilla HTML + [Tailwind CSS CDN](https://tailwindcss.com) | No build tools. Works anywhere. |
| **Logic** | Vanilla JavaScript (ES2020+) | No frameworks = nothing to break or bundle |
| **Charts** | Pure CSS/HTML (`offline-charts.js`) | No Chart.js, no canvas, fully offline |
| **Storage** | `localStorage` | Built into every browser, survives offline, zero server |
| **Offline / Install** | Service Worker + Web App Manifest | Caches all assets on first load, installable as PWA |
| **Updates** | `version.json` canary + SW messaging | User-controlled updates, no auto-reload surprises |
| **Hosting** | GitHub Pages | Free, static, no backend |
| **Tests** | Playwright | E2E tests against a local file server |

**No servers. No databases. No backend. No framework. No accounts. No trackers.**

The entire app is one `index.html` file + a service worker + a tiny charts helper. You could read every line of code in an afternoon.

## Privacy

- All data is stored in **your browser's `localStorage`** on your device
- Nothing is ever sent to any server
- There are no analytics, no crash reporters, no third-party scripts (Tailwind CDN and Google Fonts are loaded on first use only — after that, the service worker caches them locally)
- If you clear your browser/app data, your entries are gone — that's why the **Export** feature exists
